# Subscriptions Data Service
Сервис для управления подписками пользователей (CRUDL + расчет общей стоимости за период).

## Возможности
- CRUDL-операции с записями
- Расчет суммарной стоимости подписок за период с фильтрацией по пользователю и сервису
- Swagger-документация
- Логирование при помощи slog
- Миграции базы данных PostgreSQL



## Технологии
- Go 1.26
- google/uuid 1.6.0
- jackc/pgx 5.10.0
- joho/godotenv 1.5.1
- swaggo/swag 1.16.6
- PostgreSQL 18.4
- Docker/Docker Compose 4.45.0

## Конфигурация
Необходимо создать `.env`-файл в корне проекта, где указать порт запуска http-сервера сервиса, а также имя пользователя, пароль, имя хоста и базы данных в PostgreSQL. В качестве примера в проекте находится файл `.env.example`. Имя хоста базы данных должно совпадать с названием сервиса базы данных, описываемого в `docker-compose.yaml` (subscriptions-postgres).

## Использование
Сервис и его среда описаны как сервисы в рамках Docker Compose. Для удобства запуск команд можно осуществлять с помощью набора инструкций в `Makefile` и команды `make`. Команды необходимо запускать, находясь в корне проекта

### 1. Поднять сервер PostgreSQL
```
make env-up
```
Порт 5432 должен быть свободен. После запуска автоматически создастся пользователь с паролем и база данных, значения которых указаны в `.env`-файле

### 2. Применить миграции
```
make migrate-up
```
Файлы миграции хранятся в виде `.sql`-файлов в директории `migrations`

### 3. Запустить сервис
```
make service-deploy
```
Сервис будет доступен по URL `http://localhost:8080`

### 4. Сгенерировать документацию
```
make swagger-create
```
Документация хранится в директории `docs`, Swagger будет доступен по пути `http://localhost:8080/swagger/index.html`

### 5. Отправка запросов
Запросы можно отправлять с помощью `curl` или Postman.
Пример GET HTTP-запроса для получения записи о подписке по ее идентификатору:
```
curl localhost:8080/records/1
```
GET поддерживает пагинацию с помощью опциональных параметров `limit` (число записей) и `offset` (сколько с начала записей пропустить)

Данные о подписке передаются в формате JSON, пример:
```json
{
  "service_name": "Yandex Plus",
  "price": 400,
  "user_id": "60601fee-2bf1-4721-ae6f-7636e79a0cba",
  "start_date": "07-2025",
  "end_date": "08-2025"
}
```
где `end_date` – последний оплаченный месяц подписки (опционально), пустое значение трактуется как бессрочная подписка. 
Даты в параметрах запроса также должны передаваться в формате месяц-год как при передаче в JSON

### Остальные инструкции Makefile
#### Остановка сервиса
```
make service-stop
```
#### Остановка PostgreSQL
```
make env-down
```
#### Очистка данных PostgreSQL
```
make env-cleanup
```
#### Создание новой миграции
Перезаписывает `.sql`-файлы в директории `migrations`
```
make migrate-create
```
#### Откат миграций
```
make migrate-down
```
